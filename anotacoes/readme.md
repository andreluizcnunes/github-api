## ReactJS

- quando menor uma a responsabilidade de um componente melhor é.

  - Melhor para Manutenção
  - Melhor para manter o codigo
  - Melhor para expandir
  - Melhor para escalar

- Por que componentizar?

  - Manutenção mais Facil
  - Reusabeilidade
